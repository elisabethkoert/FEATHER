# `histimg` – Histology Image Object

A `histimg` object stores quantification results from one histology image set, typically a single cochlear region (e.g. one turn) imaged with a 40× oil-immersion objective.  
Quantification is produced by the **Nintendo** Arivis-based analysis pipeline described in Thirumalai _et al._ 2025 (doi:10.7150/thno.104474).

---

## Raw Data

### Directory type

The `histimg` object reads from a raw-data directory registered in the parent `anex` with `type = "NintendoRes"`.  
This directory should contain the `.csv` export files generated by the Nintendo/Arivis pipeline.

### File naming convention

Each `.csv` file corresponds to one image set. Files must include the **experiment ID** (or the animal number extracted from it) in their filename so that `listHistImgsRaw` can associate them with the correct experiment.  
Metadata encoded in the filename is used to set `histimg.side`, `histimg.turn`, and `histimg.version`:

```
<ExpID>_<Side><Turn>[-<version>].csv

Examples:
  GEK030_Rapex.csv            → side=R, turn=apex, version=1
  GEK030_Lbase_v2.csv         → side=L, turn=base, version=2
  GEK030_Lmid_v1_40x.csv     → side=L, turn=mid,  version=1
```

| Filename token | Property set | Expected values |
|---------------|-------------|-----------------|
| `L` or `R` | `histimg.side` | `L` / `R` |
| `apex`, `base`, or `mid`/`med` | `histimg.turn` | `apex` / `base` / `mid` |
| `v<n>` | `histimg.version` | integer ≥ 1 (default 1) |

### CSV file content

Each `.csv` file must be comma-delimited and contain a header row.  
The following column names are read by `readNintendoResults`:

| Column name | Required | Description |
|-------------|----------|-------------|
| `AllSGNs` | ✔ | Total number of detected spiral ganglion neurons |
| `No_Positive` | ✔ | Number of GFP-positive (transduced) neurons |
| `Volume_ROI_microm3_` | ✔ | 3D volume of the region of interest [µm³] |
| `Volume_ROI_Nintendo_Style_microm3_` | optional | Volume of manually drawn Rosenthal's canal [µm³] |
| `Volume_Slice_microm3_` | optional | Volume of a single center slice [µm³] (for 2-D density) |
| `Volume_Slice_Nintendo_Style_microm3_` | optional | Nintendo-style slice volume [µm³] |
| `numPlanesVolume` | optional | Number of optical planes in the 3-D volume (each 1 µm thick) |
| `AllSGNsSlice` | optional | SGN count in the center slice (used with `Volume_Slice_microm3_`) |
| `usedGFPThreshold` | optional | GFP threshold used during analysis |

Columns not listed above are silently ignored.

---

## Calibration / Hardware Files

The `histimg` object does **not** require any instrument-specific calibration files.  
Imaging parameters (objective, voxel size, etc.) are expected to be stored with the raw confocal image stack in a format readable by a future `readMicroscopeSettings` function (not yet implemented).

---

## SeriesID Format

The `SeriesID` passed at construction time must encode the relevant imaging metadata so that `histimg` can self-populate `side`, `turn`, and `version`:

```
<Side>_<turn>_<v<version>>_40x

Examples:
  R_apex_v1_40x
  L_base_v2_40x
  L_mid_v1_40x
```

---

## Processing Overview

| Step | Function | Description |
|------|----------|-------------|
| 1 | `listHistImgsRaw(ee)` | Scans the `NintendoRes` raw directory and builds a list of available image series |
| 2 | `histimg(ee, SeriesID, D, filename)` | Constructs the object; parses side/turn/version from SeriesID |
| 3 | `readNintendoResults(obj)` | Reads the `.csv` file and populates all density/count properties |
| 4 | `saveHistimg(obj)` | Saves the object to `processedDataDir/HISTO/H_<ExpID>_<SeriesID>.mat` |
| 5 | `chooseHistImgToUse.mlapp` | GUI for selecting the preferred image when multiple exist for the same region |

| `List_Hist_raw.mat` | `processedDataDir/HISTO/` | Cached list of raw image series IDs |

## Key Object Properties

| Property | Unit | Description |
|----------|------|-------------|
| `nCells` | count | Total number of SGNs detected |
| `nPosCells` | count | Number of GFP-positive (transduced) SGNs |
| `volume` | µm³ | 3-D ROI volume (automatic detection) |
| `volumeNintendoStyle` | µm³ | 3-D ROI volume (manual Rosenthal's canal drawing) |
| `density` | SGNs / 10⁵ µm³ | SGN density from automatic volume |
| `densityNintendoStyle` | SGNs / 10⁵ µm³ | SGN density from Nintendo-style volume |
| `transductionRate` | fraction 0–1 | `nPosCells / nCells` |
| `density2D` | SGNs / 10³ µm² | 2-D density derived from 3-D volume ÷ numPlanesVolume |
