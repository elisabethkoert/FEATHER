# `icme` – Inferior Colliculus Multielectrode Recording Object

An `icme` object represents one inferior-colliculus multielectrode recording acquired with the **ExpControl** and **Cheetah/Neuralynx** recording system.

---

## Raw Data

### File location

All raw data for one recording series must reside in a single directory registered in the parent `anex` object with `type = "IC"`.

```
<rawDataDir>/
  <SeriesID>.m                  % ExpControl log file
  <RawDataFolder>/              % Neuralynx Cheetah output folder
    *.ncs                       % continuous signal files (one per electrode)
    *.nev                       % event / TTL file
    *.nse                       % spike waveform files (optional)
    CheetahLostADRecords.txt    % Cheetah integrity log (optional)
```

The sub-folder name `<RawDataFolder>` is read from the field `ExpInfo.RawDataFolder{1}` inside the `.m` log file.

---

### ExpControl log file (`<SeriesID>.m`)

The log file is a plain MATLAB script generated automatically by ExpControl.  
When run with `run()`, it returns a struct (stored in `icme.ExpInfo`) that must contain at minimum:

| Field | Description |
|-------|-------------|
| `exp_type` | Experiment type string (e.g. `'MX_tones'`, `'OBIS_LS594_Pulse'`) |
| `stimlist` | Matrix of stimulus parameters (rows = conditions) |
| `stimheader` | Cell array of column names for `stimlist` |
| `dura` | Stimulus duration per condition [s] |
| `n_rep` | Number of repetitions per stimulus |
| `animal_ID` | Experiment ID string (must match `icme.ExpID`) |
| `RawDataFolder` | Cell array – `{1}` is the path to the Cheetah data sub-folder |
| `FreqAcqStim` | Acquisition/stimulus frequency [Hz] (used in processing) |

---

### Neuralynx / Cheetah files

| File type | Extension | Required | Description |
|-----------|-----------|----------|-------------|
| Continuous signal | `.ncs` | ✔ | One file per electrode channel; read via `Nlx2MatCSC` |
| Event / TTL | `.nev` | ✔ | Stimulus trigger timestamps; read via `Nlx2MatEV` |
| Spike waveforms | `.nse` | optional | Pre-thresholded spike waveforms from Cheetah online sorting |
| Lost-record log | `CheetahLostADRecords.txt` | optional | Data-integrity information |

> **Note:** The Neuralynx MATLAB import tools (`Nlx2MatCSC`, `Nlx2MatEV`) from the Neuralynx MatlabImportExport package must be on the MATLAB path.

## Calibration Files

Calibration is handled by `getCalibration` (stimlist already uses calibrated values from ExpControl) or `calculateCalibration` (stimlist stores raw drive percentages and FEATHER interpolates to mW).

### When is a calibration file needed?

| Experiment type | Function to call | Calibration file needed? |
|----------------|-----------------|--------------------------|
| `OBIS_*`, `OXXIUS_*`, `DarkRedLaser_*` | `getCalibration` or `calculateCalibration` | ✔ see below |
| `SBcreeLED10x1` | `calculateCalibration` | ✔ see below |
| Acoustic (`MX_tones`) | — | ✗ no file needed |

### File naming convention

Calibration files are expected in the **IC raw data directory** (default) or a custom `calibDir`.  
They are generated by the **LaserControl** software (https://gitlab.gwdg.de/optogroup/opticsmodules/lasercontrol):

```
<date>_<laserType>_<ComPort>_Power_Calibration_<OD>.txt

Examples:
  20240312_OBIS_COM10_Power_Calibration_0.txt
  20240312_OBIS_COM10_Power_Calibration_1.txt     (OD 1 filter)
  20240312_SB_COM5_Power_Calibration_ID_XYZ_ResistorNo_2.txt  (µLED)
```

The `OD` and `ComPort` parameters must match what was entered in `ICuserInput.mlapp` (stored in `ICUserInput_<ExpID>.mat` in the processed data directory).

### File content

Each calibration file contains a short header followed by a tab-delimited table.  
The header must contain the string `Power (mW)` for automatic header-detection.  
Expected columns (default names):

| Column | Description |
|--------|-------------|
| `Power_Percent` | Drive percentage to the laser (0–100 %) |
| `Current_mA` | Corresponding drive current [mA] |
| `Power_mW` | Measured optical output power [mW] |

### User-input prerequisite

The GUI `ICuserInput.mlapp` must be completed before calling any calibration function.  
It saves `ICUserInput_<ExpID>.mat` to `processedDataDir/ICME/` and records for each `icme`:

- Whether the recording should be used (`Use` flag: `1` = use, `-1` = skip)
- The **OD filter** value applied during the recording
- The **COM port** of the laser
- The **stimulus type** (from a dropdown)
- Electrode probe / position information

## Processing Overview

| Step | Function | Description |
|------|----------|-------------|
| 1 | `allIcme(ee)` | Calls `initIcme` for all ICs: loads log file → saves icme stub |
| 2 | `ICuserInput(ee)` | GUI: records hardware, filter, quality flags per icme |
| 3 | `getCalibration` / `calculateCalibration` | Reads LaserControl `.txt` file and populates `icme.C` |
| 4 | `ExtractMUAfromRawDataIntoSL` | Reads `.ncs` / `.nev` files, detects MU spikes, fills `icme.SL` |
| 5 | `saveIcme` | Saves icme to `processedDataDir/ICME/IC/IC_<ExpID>_<SeriesID>.mat` |

### Spike extraction defaults

`ExtractMUAfromRawDataIntoSL` uses the following defaults (all adjustable):

| Parameter | Default | Description |
|-----------|---------|-------------|
| `threshold` | 4 × MAD | Spike detection threshold |
| `pre_time` | 0.5 ms | Waveform window before spike |
| `post_time` | 1 ms | Waveform window after spike |
| `ref_time` | 1 ms | Refractory period |
| `low_filt` | 600 Hz | High-pass cut-off |
| `high_filt` | 6000 Hz | Low-pass cut-off |
| `filt_ord` | 4 | Butterworth filter order |
| `prestimrectime` | 150 ms | Recording window before trigger |
| `poststimrectime` | 150 ms | Recording window after trigger |
| `use_artefact_removal` | true | Subtract global mean to reduce stimulation artefacts |

## Processed Data Files

| File | Location | Description |
|------|----------|-------------|
| `IC_<ExpID>_<SeriesID>.mat` | `processedDataDir/ICME/IC/` | Saved `icme` object (raw field `R` emptied before saving) |
| `ICUserInput_<ExpID>.mat` | `processedDataDir/ICME/` | User-input table from `ICuserInput.mlapp` |
| `<ExpID>_<SeriesID>_ICME_Resort.txt` | `processedDataDir/ICME/RESORT/` | Spike-list export (CSV, generated by `ExtractMUAfromRawDataIntoSL`) |
| `*tonotopy_res_*.mat` | `processedDataDir/ICME/` | Tonotopy analysis results |
| `*_SR_*.mat` | `processedDataDir/ICME/SR/` | Spike-rate analysis results |

## Key Object Properties

| Property | Description |
|----------|-------------|
| `ExpID` | Experiment / animal ID (e.g. `'GEK111'`) |
| `SeriesID` | Recording series ID (e.g. `'GEK111_0001'`) |
| `ExpInfo` | Struct from the ExpControl log file |
| `Stim` | Stimulus parameters extracted from `ExpInfo` |
| `SL` | Spike-list struct (filled by `ExtractMUAfromRawDataIntoSL`) |
| `C` | Calibration struct (filled by `getCalibration` / `calculateCalibration`) |
| `D` | Raw data directory struct |
